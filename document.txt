================================================================================
  Slay The Robot（Slayword）— 项目文档
================================================================================

一、项目概述
--------------------------------------------------------------------------------

本项目是一款基于 Godot 4.6 开发的单机 Roguelike 卡牌构筑游戏，玩法骨架接近《杀戮尖塔》类作品：
玩家在程序化生成的地图上选择路线，经历战斗、商店、休息、事件等节点，通过构筑牌组、收集遗物与消耗品，
最终达成通关目标。

与众不同的是：游戏将「英语词汇学习」深度嵌入战斗循环——出牌前可能触发词汇复习（拼写、释义、四选一、
英文回忆自评等），并与简易间隔复习（SRS）、多词书合并等功能结合，形成「边玩边背」的体验。

【AI 在本项目中的定位（强调）】
  本项目的 AI（OpenAI 兼容大语言模型 API）不承担战斗数值或关卡生成，而是专注于「词汇学习体验」：在玩家
  已导入或启用的词书基础上，按需生成带语境的英文例句与中文释义辅助（见 VocabStudy、UserSettingsData）。
  换言之：Roguelike 玩法由本地数据与引擎驱动；AI 负责把「词条列表」升级为「可在复习界面直接阅读的、
  贴合领域偏好的例句材料」，并与缓存、预取策略配合，尽量避免拖累出牌节奏。

  AI 两点核心作用（务必记住）：
    （1）语境化学习内容：针对 headword、释义、词书乃至用户选择的领域标签（如考研风、游戏科幻、日常口语等），
        动态生成多条英文例句及对应中文整句翻译/gloss，让用户在出牌门控复习时看到的是「活句子」而非仅有
        单词卡片，从而强化理解与记忆线索。
    （2）体验与成本的工程化平衡：通过 user:// 例句缓存、开局/地图节点触发的后台批量预生成、出牌按需补拉、
        以及可选的离线脚本批量生成，在无 Key 时仍可完全依赖词书题干游玩；有 Key 时则减少重复请求与界面卡顿。

引擎：Godot 4.6（渲染 GL 兼容模式，便于跨平台）
主场景：res://scenes/Root.tscn
默认分辨率：1280×720，窗口默认不可拉伸（可在工程中调整）


二、核心玩法与流程
--------------------------------------------------------------------------------

【开局与存档】
  · 标题界面：继续游戏、放弃回合、新游戏、图鉴、背单词偏好、设置、退出。
  · 新开局：选择角色，查看初始生命、金币、描述与起始遗物；可输入种子；可选自定义难度 / 开局修饰符
    （Run Modifier）。
  · 回合进行中可暂停：恢复、返回标题、放弃本局。

【地图与节点】
  · 程序化地图：玩家在可选路径上的节点间移动（地图 UI、路线连接等）。
  · 典型节点类型包括战斗、精英、首领、商店、休息点、宝箱、事件等（具体由 Act / 事件池数据驱动）。

【战斗】
  · 回合制：玩家回合（出牌、使用消耗品、结束回合）与敌人回合交替。
  · 能量与抽牌：经典攻防牌库循环（抽牌堆、弃牌堆、消耗堆、洗牌等）。
  · 目标选择：需要指定目标的卡牌在进入队列前选择敌人。
  · 状态效果：buff/debuff、意图显示、多层生命条表现等。
  · 胜利后可进入奖励界面（金币、卡牌三选一、遗物等，依规则而定）。

【牌组构筑】
  · 卡牌具有类型、稀有度、颜色（流派）、能量消耗（含可变费用 X）、关键词说明等。
  · 支持升级、变换、从弃牌/抽牌堆检索、复制、额外打出等进阶机制。
  · 手牌上限、保留（Retain）、放逐（Banish/Exhaust）等与尖塔-like 设计一致方向的规则。

【遗物（Artifacts）】
  · 被动效果贯穿战斗或地图，例如每杀怪抽牌、改变费用规则等。
  · 词汇相关：如「初稿免责」类遗物——每场战斗第一次答错或「看答案」时可免除 SRS 失败惩罚并仍允许
    打出该张牌（具体以数据为准）。

【消耗品（Consumables）】
  · 战斗中可使用或丢弃，拓展单次战斗的策略空间。

【商店 / 休息 / 宝箱 / 事件】
  · 商店：购买卡牌、遗物、消耗品等（具体条目由数据定义）。
  · 休息：可选择回血或升级卡牌等（由 RestAction 数据驱动）。
  · 宝箱与事件：通过 Dialogue / Event 数据触发分支与奖励。


三、背单词系统（与战斗绑定）
--------------------------------------------------------------------------------

【出牌门控】
  · 在战斗中，玩家打出卡牌前会经由 VocabStudy.gate_before_play()：随机抽取词条弹出复习界面
    （WordReviewOverlay）。
  · 复习结果影响是否允许本次出牌：答对则正常出牌；跳过 / 答错则可能阻断出牌并更新学习记录。

【题型与流水线】
  · 出牌复习可在多种模式中随机（用户设置）：拼写（spell）、看英选中译（meaning）、四选一（mc4）、
    英文提示自评 recall（记得/不记得）等。
  · 新词学习可按固定流水线环节勾选：英译中、中译英、拼写、听写（dictation）等（见 UserSettingsData）。

【词书与合并池】
  · 支持多词书合并为题库：内置默认词表、打包词书（如 NETEM 考研 5530）、用户导入 JSON 词书。
  · 词书路径：res://data/vocab_books/*.json、user://vocab_books/*.json；格式说明见 data/vocab_books/
    README_IMPORT.txt。
  · 启用列表由 settings_vocab_enabled_book_ids 控制；词条 id 会带 book_id 前缀避免冲突。

【间隔复习与每日配额】
  · 简易 SRS：根据作答结果更新词条记忆状态。
  · 支持「每日新词数量」「每日到期复习上限」等，控制学习与复习压力。
  · 自然日维度：按日期轮换「到期」词的参与上限，避免同一天刷爆。

【例句与 AI API（可选，上文「AI 两点」的具体落地）】
  · 若配置 OpenAI 兼容 API（环境变量、res://.env、DOTENV_PATH 或用户设置中的 Key / Base URL / Model），
    可为词条批量或按需生成英文例句及中文翻译，并缓存到 user://vocab_example_cache.json——这是 AI 作用（1）
    的数据来源；（2）则由缓存键、双 HTTPRequest 避免并发 ERR_BUSY、每日顺序预算与后台预取等实现。
  · 支持领域偏好标签（游戏科幻、日常、考研风等），与开局设置中的勾选一致；提示词在 VocabStudy 内针对
    「单条生成」与「批量生成」分别定义，以保证 JSON 结构与游戏解析一致。
  · 开局或地图节点切换等时机可触发后台预取例句，减轻出牌时等待。
  · 仓库可提供离线脚本 tools/generate_vocab_examples.py（与 .env 约定一致）用于批量生成（同属 AI 扩展学习
    材料，但不强制联网游玩）。


四、技术架构亮点
--------------------------------------------------------------------------------

【数据驱动与 Schema】
  · Global.gd 维护统一 SCHEMA：卡牌、敌人、遗物、事件、地图 Act、关键词、颜色、修饰符、自定义 UI、
    自定义信号等均以 SerializableData + JSON 加载。
  · register_rod() 按类型注册只读数据，便于测试数据注入与模组扩展。

【动作系统】
  · ActionHandler + ActionGenerator：卡牌效果、敌人行为、地图交互均拆解为 BaseAction 队列执行，
    支持拦截器（ActionInterceptor）在处理链中改写或监听（如双倍出牌、状态衰减拦截等）。
  · 与杀戮尖塔-like 的「效果原子化」一致，便于扩展复杂联动。

【信号总线】
  · Signals 单例集中声明全局信号（回合、卡牌、地图、奖励、词汇例句 API 进度等），模块解耦。

【国际化】
  · I18N 单例 + Root.gd 内对各界面节点的静态翻译键映射；默认语言 zh_CN，可扩展其它 locale。

【模组 / 外部数据】
  · external/mod_info.json 声明各 external/data/* 目录对应的 Data 类型，可在不改动核心代码的情况下
    以 JSON 追加或覆盖内容。
  · external/data/_templates/README.md 说明安全模板与 --export-test-data 导出工作流。

【移动端与触控】
  · project.godot 中可锁定横竖屏、启用鼠标模拟触摸等；SlayMobileStyle 提供触控友好的面板与按钮样式常量。
  · 词汇导入支持桌面绝对路径与用户目录 JSON，移动端可通过文件选择器读 UTF-8 文本后导入。

【调试】
  · DebugLogger 单例；部分卡牌含调试用途（如 card_debug_log）。

六、适用人群与使用场景
--------------------------------------------------------------------------------

  · 喜爱 Roguelike 卡牌、希望能在反复开局中巩固英语词汇的玩家。
  · 需要考研 / CET / 自定义词表的用户：通过 JSON 导入或启用打包词书合并学习。
  · 希望在离线环境下游玩、仅在愿意时配置 API 生成例句的用户。

八、项目亮点（总结）
--------------------------------------------------------------------------------

  1. 「卡牌 Roguelike × 词汇 SRS」闭环：出牌前复习直接挂钩战术节奏，学习不是额外迷你游戏。
  2. 【AI】语境化学习：大模型按词条与领域偏好生成例句与中译材料（见第三节「AI 两点」），显著提升背诵界面
     信息量；非战斗逻辑，不影响关卡公平性。
  3. 【AI】可关断的工程管线：缓存 + 预取 + 按需请求 + 可选离线脚本，降低卡顿与 token 消耗，无 API Key 仍可玩。
  4. 多词书合并 + 用户导入：兼顾内置、大包考研词表与自定义词书；词条 id 带命名空间防冲突。
  5. 可配置的复习题型与学习流水线，适应不同记忆习惯。
  6. 深度数据驱动：卡牌、敌人、事件、遗物、修饰符等均可通过 JSON 模组扩展。
  7. 成熟的动作队列与拦截器架构，便于扩展复杂卡牌与遗物逻辑。
  8. 国际化与触控/移动端取向的工程配置，便于多端发布。
