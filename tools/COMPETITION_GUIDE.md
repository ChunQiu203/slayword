# Slay The Robot (Slayword) — vivo 竞赛演示指南

## 一、作品简介

「Slay The Robot」是一款将 **Roguelike 卡牌战斗 × 英语词汇间隔复习 × AI 大模型语境化学习** 深度整合的移动端应用。玩家在类《杀戮尖塔》的卡牌构筑玩法中，出牌前需要完成词汇拼写/释义复习——答对才能出牌，答错被拦截。AI 大模型为每个词汇动态生成贴合领域偏好的英文例句与中文翻译，让单词记忆不再枯燥。

### 竞赛赛道
手机 AI 助手 × 移动学习场景

### 技术栈
- **引擎**: Godot 4.6 (GL Compatibility)
- **语言**: GDScript
- **AI 能力**: OpenAI 兼容 API + vivo 蓝心端侧大模型适配层
- **SRS 算法**: SM-2 间隔复习变体
- **数据驱动**: JSON 模组系统，支持自定义词书导入

## 二、演示流程（8 分钟核心演示）

### 阶段 1: 开场（30秒）
1. 展示标题画面，说明游戏核心理念："边玩 Roguelike 卡牌边背单词"
2. 进入「背单词偏好」页面，展示已启用的词书（内置词库 + 考研词书 5530 词）
3. 展示 AI 例句领域偏好设置（游戏科幻 / 日常口语 / 考研真题风）
4. 展示例句生成进度

### 阶段 2: 新游戏 - 选角（30秒）
1. 选择「纸间行者」（词汇学者型角色）
2. 介绍角色特色：起始遗物可预知抽牌堆，专属词汇联动卡牌

### 阶段 3: 首次战斗 - AI 例句展示（2分钟）
1. 进入战斗，展示地图路线
2. 打出第一张攻击牌 → 触发词汇复习弹窗
3. **关键展示点**: 
   - 新词「学习面板」：展示 AI 生成的例句（英文例句 + 中文翻译 + 义项提示）
   - 学习流水线：英译中选义 → 中译英四选一 → 拼写
   - 答对 → 卡牌打出，展示 SRS 下次复习时间预测
4. 展示后台预取的 AI 例句缓存机制

### 阶段 4: 战斗深度展示（2分钟）
1. 展示不同卡牌类型：攻击/技能/能力
2. 展示能量管理、格挡、敌人意图系统
3. 展示腐蚀/Bomb 等状态效果
4. 展示「学以致用」「知识壁垒」等词汇联动卡
5. 击败敌人，获得卡牌选秀奖励

### 阶段 5: AI 增强功能（2分钟）
1. 暂停菜单 → 展示 AI 例句生成设置
2. 切换例句领域偏好（如从"游戏科幻"切换到"考研真题"）
3. 展示例句风格变化
4. 说明离线降级策略（无网络时完全依赖本地词书）
5. 展示词书导入功能（可现场演示导入一个自定义词书）

### 阶段 6: 收尾（1分钟）
1. 展示图鉴系统（所有卡牌/遗物/敌人可浏览）
2. 总结核心创新：
   - 游戏化学习：出牌前复习直接挂钩战术节奏
   - AI 语境化：大模型生成自然例句，显著提升记忆效果
   - 端侧适配：支持 vivo 蓝心大模型，保护用户隐私
   - 多词书合并：支持 7 套主流考试词书 + 自定义导入

## 三、AI 系统架构

### 3.1 双引擎设计
```
┌─────────────────────────────────────────┐
│              VocabStudy 单例             │
│  ┌─────────────┐  ┌───────────────────┐ │
│  │ 云端 API 引擎 │  │ 端侧模型引擎(蓝心) │ │
│  │ OpenAI 兼容  │  │ ON-DEVICE适配层   │ │
│  └──────┬──────┘  └────────┬──────────┘ │
│         │                  │            │
│  ┌──────┴──────────────────┴──────────┐ │
│  │     例句缓存 (user:// JSON)         │ │
│  │     + 后台预取 + 按需补拉           │ │
│  └────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

### 3.2 三级例句策略
1. **每日顺序批量**: 开局后按词池顺序，每天处理 N 个词条
2. **后台预取**: 地图节点切换/战斗结束等时机，预取即将复习的词条
3. **按需补拉**: 出牌前如遇缺例句词条，实时请求

### 3.3 端侧适配
- 环境变量 `ONDDEVICE_MODEL_ENDPOINT` + `ONDDEVICE_MODEL_NAME` 配置
- `ONDDEVICE_EXAMPLE_PREFER=1` 优先端侧
- 自动 fallback 到云端 API
- 无网络时完全离线运行（依赖本地词书 + 例句缓存）

## 四、技术亮点

### 4.1 数据驱动架构
- 所有游戏内容（卡牌/敌人/遗物/事件/角色）通过 JSON 定义
- 支持模组扩展，玩家可自定义游戏内容
- `Global.SCHEMA` 统一管理 21 种数据类型

### 4.2 动作队列 + 拦截器系统
- ~50 种原子化动作类型
- 10 个拦截器实现复杂状态联动
- 所有游戏效果（伤害/格挡/状态/抽牌）通过同一管线执行

### 4.3 智能出题策略
- 根据 SRS 到期优先级 + 每日配额选择出题词条
- 多种题型随机：拼写/释义/四选一/自评回想
- 学习流水线：英译中→中译英→拼写→听写
- 错误模式追踪（拼写错 vs 释义混淆）

### 建议预设种子
在游戏开始前设置 `run_seed = 42` 可确保稳定的地图生成和敌人配置。

## 六、离线降级方案

比赛现场如遇网络问题：
1. 完全离线启动游戏
2. 依赖 `user://vocab_example_cache.json` 中预缓存的例句
3. 内置词书仍然提供完整的拼写/释义复习
4. 游戏核心玩法不受影响
